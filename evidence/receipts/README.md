# Receipts

This directory is reserved for cryptographically bound execution, repair, witness, and reproduction receipts.

A receipt must be interpreted only with its referenced canonicalization profile, governance bundle, artifact hashes, environment identity, challenge set, signer identity, and certificate lifecycle state.

Cryptographic validity does not establish semantic correctness, witness independence, regulatory approval, or production authority.
